Fix Strategy: Zombie Resources

Root Cause

cleanup.py calls find_cleanup_candidates() which uses k8s_client (a direct kubeconfig connection from EC2). This connection is permanently dead — the cluster is only reachable from inside the cluster. The agent handles all cluster reads.

Fix

Rewrite the zombie-resources endpoint (and all /v1/cleanup/* sub-routes) to derive candidates from data already in PostgreSQL via db_manager:

Zombie typeAgent data sourceDetection rule
Zombie Podspods.itemsNo owner_kind AND phase != Running AND age > 7d
Completed/Failed Jobsworkloads.jobs.itemssucceeded > 0 OR failed > 0 AND age > 7d
Old ReplicaSetsworkloads.replicasets.orphanedAlready pre-filtered by agent (replicas=0, owned by Deployment)
Unattached PVCsstorage.orphaned_pvcsPre-filtered by agent (phase != Bound)
Services no endpointsnetwork.services.itemsendpoints_count == 0 AND age > 14d AND not kube-system
Idle Namespacespods.items + workloadsNamespace has no running pods
